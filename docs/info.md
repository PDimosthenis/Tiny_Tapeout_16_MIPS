# 16-bit MIPS Single Cycle Processor

This project implements a small 16-bit MIPS-like single-cycle CPU for Tiny Tapeout.

## Overview

The CPU executes 16-bit instructions, but instructions are loaded through `uio_in[7:0]` while control/address are driven by `ui_in`.

## Main Features

- 16-bit datapath and 16-bit registers
- 16-bit instruction words
- 4 x 16-bit register file
- 2-word 16-bit data memory
- Single-cycle execution
- Load/run mode controlled through `ui_in[7]`

## Pin Usage

- `ui_in[7]`: mode select, `0` = load, `1` = run
- `ui_in[6]`: byte select, `0` = low byte, `1` = high byte
- `ui_in[2:0]`: instruction address during load mode
- `uio_in[7:0]`: instruction byte during load mode
- `uo_out[7:0]`: lower 8 bits of ALU result in run mode
- `uio_out[7:0]`: upper 8 bits of ALU result in run mode
- `uio_oe[7:0]`: output enable mask (`0x00` load, `0xFF` run)

## Load Flow

To load one 16-bit instruction:

1. Set `ui_in[7] = 0` for load mode.
2. Put the target instruction address on `ui_in[2:0]`.
3. Write the low byte on `uio_in[7:0]` with `ui_in[6] = 0`.
4. Write the high byte on `uio_in[7:0]` with `ui_in[6] = 1`.
5. Switch `ui_in[7] = 1` to run the CPU.

## Notes

- The register file uses 4 architectural registers (2-bit register indices).
- R-type field mapping is `rs=instruction[11:10]`, `rt=instruction[9:8]`, `rd=instruction[7:6]`.
- I-type field mapping is `rs=instruction[11:10]`, `rt=instruction[9:8]`, `imm=instruction[7:0]`.
- Jump target is `instruction[2:0]`.
- `MemRead`, `MemWrite`, `RegWrite`, `ALUsrc`, `RegDst`, `MemtoReg`, and `Jump` are control signals generated by the control unit to steer the datapath.
